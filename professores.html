Entidade Professores (Instructors)

Assim como Students (Alunos), os professores tem como base a entidade User

Para identificarmos que um usuário é um professor, deve se olhar o campo entity que armazena o tipo de usuário em forma de string, no caso do professor, este campo deve estar como instructor, e é com esse nome que irei tratá-lo a frente.

Portanto, Instructor é representado através da seguinte entidade:

Operações HTTP

A URL para acessar o recurso (entidade) instructor é: https://hackjoy-api.herokuapp.com/instructors

Novo instructor


Método: POST
URL: https://hackjoy-api.herokuapp.com/instructors/new

Para se cadastrar um novo instructor é necessário informar os seguintes dados:

  • name: Nome do instructor
  • birth_date: Data de nascimento
  • nationality: País de nacionalidade
  • document: Documento. Se o país de nacionalidade for o Brasil, o documento exigido é CPF, caso contrário, pode ser inserido qualquer outro tipo de documento que identifique a pessoa.
  • gender: Genêro com o qual se identifica. Portanto, há 3 opções que podem ser colocadas: Feminino, Masculino ou Outra
  • email: E-mail. Este e-mail será usado como chave única dentro da plataforma, logo não haverá mais de um usuário com o mesmo email.
  • password: Senha
  • about: Este dado é a descrição do instructor que está sendo cadastrado.

Autenticar instructor


Método: POST
URL: https://hackjoy-api.herokuapp.com/instructors/login

Para que um instructor consiga fazer login e ter acesso aos serviços do sistema, deverá ser informado os seguintes dados:

  • email: E-mail cadastrado
  • password: Senha cadastrada

Ao enviar a requisição, será verificado se os dados batem e se o status do usuário está como "Active" no banco de dados. Caso seja validado corretamente, o sistema irá retornar um token. Este token deverá ser enviado no header de requisições que necessitem de autenticação.

Listar

Método: GET URL: https://hackjoy-api.herokuapp.com/instructors

Retorna um JSON com todos os professores cadastrados

Encontrar instructor por id

Método: GET URL: https://hackjoy-api.herokuapp.com/instructors/{id}

Retorna o instructor que possui o id informado por parâmetro.

Importante: Todas as rotas que exigerem id, será passado o id da entidade instructor e não dá entidade user.

Excluir instructor por id

Método: DELETE URL: https://hackjoy-api.herokuapp.com/instructors/{id}

Delete o instructor que possui o id informado por parâmetro.

Atualizar instructor por id

Método: PUT URL: https://hackjoy-api.herokuapp.com/instructors/{id}

Altera a entidade user e sua entidade instructor que possui o id informado por parâmetro.

Para se atualizar um instructor é necessário informar os seguintes dados:

  • name: Nome do instructor
  • birth_date: Data de nascimento
  • nationality: País de nacionalidade
  • document: Documento. Se o país de nacionalidade for o Brasil, o documento exigido é CPF, caso contrário, pode ser inserido qualquer outro tipo de documento que identifique a pessoa.
  • gender: Genêro com o qual se identifica. Portanto, há 3 opções que podem ser colocadas: Feminino, Masculino ou Outra
  • email: E-mail. Este e-mail será usado como chave única dentro da plataforma, logo não haverá mais de um usuário com o mesmo email.
  • about: Este dado é a descrição do instructor que está sendo alterado.
