Entidade Alunos (Student)

Antes de falar detalhadamente sobre Alunos, é necessário entender que no sistema há três tipos de usuários: Alunos, Professores e Administradores. Todos eles possuem dados em comum, portanto no banco de dados os referenciamos com a mesma entidade

Para identificarmos que um usuário é um Aluno, deve se olhar o campo entity que armazena o tipo de usuário em forma de string, no caso do aluno, este campo deve estar como student, e é com esse nome que irei tratá-lo a frente.

O problema é que não basta que somente haja uma entidade (no caso user) para representar os três usuários ao mesmo tempo. É fato que eles possuem muitos dados em comum, e é por isso que user é usado como uma entidade principal, porém eles vão ter dados específicos. Por exemplo, um professor vai ter dados sobre sua descrição, student vai ter dados como seu Elo. Esses dados não podem ser colocados todos na mesma entidade (user), portanto há uma segunda entidade representativa para os usuários. No caso do student, se usa esta entidade para especificá-lo:

Operações HTTP

A URL para acessar o recurso (entidade) student é: https://hackjoy-api.herokuapp.com/students

Novo student


Método: POST
URL: https://hackjoy-api.herokuapp.com/students/new

Para se cadastrar um novo student é necessário informar os seguintes dados:

  • name: Nome do student
  • birth_date: Data de nascimento
  • nationality: País de nacionalidade
  • document: Documento. Se o país de nacionalidade for o Brasil, o documento exigido é CPF, caso contrário, pode ser inserido qualquer outro tipo de documento que identifique a pessoa.
  • gender: Genêro com o qual se identifica. Portanto, há 3 opções que podem ser colocadas: Feminino, Masculino ou Outra
  • email: E-mail. Este e-mail será usado como chave única dentro da plataforma, logo não haverá mais de um usuário com o mesmo email.
  • password: Senha
  • elo: Este dado é o Elo do student. Ele não precisa ser informado, pois sempre que um student se cadastrar seu Elo automaticamente será 0. Só está aqui para fins de exemplo.

Autenticar student


Método: POST
URL: https://hackjoy-api.herokuapp.com/students/login

Para que um student consiga fazer login e ter acesso aos serviços do sistema, deverá ser informado os seguintes dados:

  • email: E-mail cadastrado
  • password: Senha cadastrada

Ao enviar a requisição, será verificado se os dados batem e se o status do usuário está como "Active" no banco de dados. Caso seja validado corretamente, o sistema irá retornar um token. Este token deverá ser enviado no header de requisições que necessitem de autenticação.

Listar

Método: GET URL: https://hackjoy-api.herokuapp.com/students

Retorna um JSON com todos os alunos cadastrados

Encontrar student por id

Método: GET URL: https://hackjoy-api.herokuapp.com/students/{id}

Retorna o student que possui o id informado por parâmetro.

Importante: Todas as rotas que exigerem id, será passado o id da entidade student e não dá entidade user.

Excluir student por id

Método: DELETE URL: https://hackjoy-api.herokuapp.com/students/{id}

Delete o student que possui o id informado por parâmetro.

Atualizar student por id

Método: PUT URL: https://hackjoy-api.herokuapp.com/students/{id}

Altera a entidade user e sua entidade student que possui o id informado por parâmetro.

Para se atualizar um student é necessário informar os seguintes dados:

  • name: Nome do student
  • birth_date: Data de nascimento
  • nationality: País de nacionalidade
  • document: Documento. Se o país de nacionalidade for o Brasil, o documento exigido é CPF, caso contrário, pode ser inserido qualquer outro tipo de documento que identifique a pessoa.
  • gender: Genêro com o qual se identifica. Portanto, há 3 opções que podem ser colocadas: Feminino, Masculino ou Outra
  • email: E-mail. Este e-mail será usado como chave única dentro da plataforma, logo não haverá mais de um usuário com o mesmo email.
